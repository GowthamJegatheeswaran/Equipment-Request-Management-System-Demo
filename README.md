Objectives

- To automate the equipment request process
- To reduce delays and manual paperwork
- To improve transparency in approval and allocation
- To maintain proper records of equipment usage
--------------------------------------------------------------------------
Scope

- maintains concurrent access
- User authentication and role-based access
- Equipment request submission by users
- Approval and rejection workflow by authorized staff
- Status tracking of requests
- Eligiblity of requesting multiple equipment in one request
--------------------------------------------------------------------------
Target Users

- Students
- Technical Officer
- Lecturers
- Heads of the Department
- Admin(System Manager)
--------------------------------------------------------------------------

Technology Stack
(Subject to finalization)

- Frontend: HTML, CSS, JavaScript 
- Backend:  Spring Boot,Java
- Database: MySQL / PostgreSQL
- Version Control: Git & GitHub
--------------------------------------------------------------------------

Project Management

- Methodology: Scrum
- Sprint Duration: 1–2 weeks
- Project activities are planned and tracked using Scrum methodology through Jira Software, while GitHub is used for version control and collaborative development.
--------------------------------------------------------------------------
Team Structure

Repository Structure

- /docs        → Project documentation (SRS, diagrams, reports)

--------------------------------------------------------------------------
Contribution Guidelines

- All contributors must work on separate branches
- Direct pushes to the main branch are restricted
- Changes must be submitted via Pull Requests
